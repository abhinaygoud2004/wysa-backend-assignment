8. BIG-PICTURE SUMMARY
========================================================================

The services layer contains the business logic:
- validating modules, questions, and options
- moving users through flows
- switching modules
- preserving history
- enforcing active state
- handling checkpoints
- implementing back navigation

The models layer defines how data is stored:
- `Module` stores the flow graph
- `UserState` stores current active position in a module
- `ConversationHistory` stores permanent answer history

The tests prove that the assignment behaviors work:
- starting modules
- answering questions
- switching modules
- stale links
- checkpoints
- defensive validation
- history preservation
- bonus back navigation
- completed flow handling
